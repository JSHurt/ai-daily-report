Langflow — 可视化构建与部署 AI Agent
⭐ ~152K
是什么：开源可视化 AI Agent / 工作流构建与部署平台——拖拽组件把 LLM、工具、记忆、向量库串成可运行流程，覆盖 multi-agent、RAG、API 暴露；面向「从草图到可上线服务」而不是只画架构图。
解决什么：写 Agent 最痛的是胶水代码：接模型、写工具调用、调重试、做流式输出、再包一层 API。Langflow 用 React Flow 画布把这些变成可复用节点，改完即可部署成服务，降低「只会 demo、上不了线」的落差。
适用场景：产品/算法快速验证 Agent 链路；中小团队要可维护的可视化编排；需要把工作流一键暴露为 API 给业务系统调用的场景。
怎么用：Docker / pip 安装后打开 Web UI → 从模板或空白画布搭流程 → 配置模型与工具 → 发布为 endpoint；本周活跃推送，最新正式版 v1.10.2。详见 langflow.org。
🔥 近期亮点：v1.10.2（7/7）；全局变量模型覆盖；本周持续推送（7/15）

LiteLLM — 100+ 模型统一 AI 网关
⭐ ~54K
是什么：生产向 LLM 网关（SDK + Proxy）——用 OpenAI 兼容格式统一调用 100+ 供应商（OpenAI / Anthropic / Azure / Bedrock / Vertex / vLLM / NIM…），内置计费追踪、负载均衡、护栏与日志。
解决什么：业务一旦接第二家模型，SDK 分叉、鉴权分叉、账单分叉立刻爆炸。LiteLLM 把「换模型 / 限流 / 回退 / 记成本」收成一层网关：应用只认一个 endpoint，运维在网关层做路由与可观测接入。
适用场景：多模型混部与成本优化；把自建 vLLM/Ollama 和云 API 统一入口；需要 guardrails、预算控制、团队密钥托管的中台团队。
怎么用：pip install litellm 或跑官方 Proxy Docker → 配 model_list 与密钥 → 客户端指向网关的 OpenAI base_url；可对接 Langfuse 等日志后端。最新 v1.92.0（7/12），今日仍在活跃推送。
🔥 近期亮点：v1.92.0；AI Gateway + 成本追踪；Docker 镜像 cosign 签名校验

⭐ ~31K

Langfuse — LLM 可观测与评测平台
⭐ ~31K
是什么：开源 AI 工程平台——LLM 链路追踪、指标、评测、Prompt 管理、Playground、数据集；对接 OpenTelemetry / LangChain / OpenAI SDK / LiteLLM 等，可云可自托管（YC W23）。
解决什么：Agent/工作流上线后最怕「黑盒」：慢在哪、贵在哪、哪次工具调用翻车、Prompt 改完有没有回退。Langfuse 把 traces、scores、prompt 版本收成一套工程面板，让优化有数据而不是靠感觉。
适用场景：RAG/Agent 质量回归；多团队共享 Prompt 版本；要自建 LLMOps（观测+评测+数据集）而不是只接云厂商 dashboard 的团队。
怎么用：Docker/K8s 自托管或使用云版 → 在应用侧加 SDK/OTel exporter → 看 traces、建 eval、管 prompt。最新 v3.214.0（7/15），持续高频发版。

⭐ ~36K
是什么：面向 Agents & Generative UI 的前端技术栈——React / Angular / Mobile / Slack 等，并推动 AG-UI 协议：让 Agent 不只回一段文字，而是驱动可交互界面（表单、表格、步骤条、人机共驾）。
解决什么：后端 Agent 写好了，产品侧还卡在「右侧聊天气泡」。CopilotKit 提供可嵌入的 copilot 组件与 Generative UI 模式，把 Agent 状态映射成真正的产品交互，而不是再包一层 iframe 聊天窗。
适用场景：SaaS 要内嵌 AI 副驾驶；需要人机确认/可编辑生成结果的工作流；多端（Web/移动/协作工具）统一 Agent 前端体验的团队。
怎么用：在 Next/React 项目安装 CopilotKit → 接后端 Agent runtime → 用组件挂载侧边栏/内联 copilot 与 Generative UI actions。最新 v1.63.0（7/15）。文档 docs.copilotkit.ai。
