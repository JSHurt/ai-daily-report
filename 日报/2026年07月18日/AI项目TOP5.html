Firecrawl — 给 Agent 的网页搜索与抓取 API
⭐ ~152.4K
是什么：面向 AI 的网页数据层——提供搜索、抓取、批量爬取与「和网页交互」能力，把站点内容转成干净 Markdown/结构化数据，方便 Agent 与 RAG 直接消费（官网：The API to search, scrape, and interact with the web at scale）。
解决什么：Agent 需要「上网查资料」时，自写爬虫要处理反爬、JS 渲染、分页与清洗；Firecrawl 把「搜 → 抓 → 结构化」做成统一 API，让模型专注推理与决策，而不是和 HTML 搏斗。
适用场景：竞品/行业调研 Agent；需要实时网页上下文的 RAG；自动化监控公开页面变化；给 Coding/Research Agent 补「可读网」能力。
怎么用：注册 firecrawl.dev 拿 API Key → SDK/HTTP 调用 search、scrape、crawl；也可对接常见 Agent 框架。本周仓库仍在活跃推送（7/17 有 API/计费相关提交）。文档见 firecrawl.dev。
🔥 近期亮点：~152K⭐；Search/Scrape/Crawl 一体；本周持续推送；Agent 读网首选基建

⭐ ~74.9K
是什么：文档智能解析引擎——把 PDF、Office 等复杂版式文档转成 LLM 友好的 Markdown/JSON，处理多栏、表格、公式、图片等「脏文档」结构（Transforms complex documents into LLM-ready markdown/JSON）。
解决什么：企业知识库里最多的是扫描 PDF 与排版混乱的报告；直接丢给模型会丢表格、乱公式、断段落。MinerU 把「文档 → 干净上下文」工程化，是 RAG/Agent 读本地资料前的关键预处理层。
适用场景：企业知识库/合同/研报入库；学术论文与技术手册结构化；给 Deep Research / 客服 Agent 喂高质量文档块；与 Firecrawl（网）形成「网 + 档」双源输入。
怎么用：按 opendatalab.github.io/MinerU 安装（pip/Docker）→ 对 PDF/Office 跑解析 → 输出 Markdown/JSON 进向量库或 Agent 上下文；最新发布线 3.4.4（7/10），仓库本周仍在维护。
