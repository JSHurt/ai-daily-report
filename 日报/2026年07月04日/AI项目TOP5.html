Hermes Agent
⭐ 208,688
是什么：Nous Research出品的开源AI Agent框架，本周发布v0.18.0 "The Judgment Release"——史上最大规模更新。累计1720次提交、998个合并PR、22万行新增代码、关闭949个issue。核心升级包括：全新的"判定"决策引擎、多模型仲裁机制、原生工具链深度集成。

适用场景：需要Agent做出关键决策的场景（代码审查、安全审计、金融分析）；已有Agent部署但缺乏可靠决策机制的团队；追求高可靠性AI自动化的企业。
怎么用：uv tool install hermes-agent，更新到v0.18.0后配置判定策略，Agent在关键节点自动触发多模型仲裁。

Omnigent
⭐ 6,051
是什么：开源多Agent元编排框架。v0.4.0（7月3日发布）将harness体系升级为真正的插件平台——任何编码Agent都可以作为可安装的npm/uv包接入。一周之内从"集成7个agent harness"进化到"任何人都可以写自己的agent插件，发布到生态"。
解决什么：不同Agent工具之间切换割裂，技术栈被锁定在特定供应商。Omnigent将Agent本身变成可插拔的组件，用户自由组合最优Agent处理不同任务——同一个终端里，Hermes Agent写代码、Qwen Code审阅、Kiro补全——做到真正的"Agent自由市场"。
适用场景：重度AI编码用户需要多Agent协作；团队希望统一管理不同Agent的API配额和任务分配；想开发自己Agent插件并接入统一编排平台的开发者。
怎么用：uv tool install omnigent==0.4.0，配置各Agent的API key，通过统一CLI向不同Agent分派任务。开发者可通过插件SDK创建自己的harness包。
🔥 本周重大升级：7月3日v0.4.0，Agent harness变成插件平台——任何人都可发布Agent插件

Browser Use
⭐ 102,498
是什么：让AI Agent操控浏览器的开源引擎。v0.13.3（7月1日发布）带来三项重大升级：CLI 3.0（基于Browser Harness的全新命令行）、browser-use skill（编码Agent可直接安装的skill包）、Browser Harness 0.1.4稳定版。意味着Agent不再需要复杂的Selenium/Playwright集成，一句话就能操控浏览器。
解决什么：AI Agent与Web交互始终是痛点——网页抓取不稳定、动态渲染困难、登录态管理复杂。Browser Use提供统一的浏览器自动化层，Agent通过自然语言描述意图即可完成网页操作，底层由Browser Harness保证可靠性。
适用场景：需要Agent自动完成网页操作的任务（数据采集、表单填写、电商比价）；RAG系统需要动态抓取最新网页内容；测试自动化场景。
怎么用：pip install browser-use==0.13.3，在Agent代码中import browser_use，一行browser.use("打开GitHub搜索hermes-agent")即可。
🔥 本周重大发布：7月1日v0.13.3，CLI 3.0 + Browser Harness稳定版 + 可安装为Agent skill

Headroom
⭐ 55,832
是什么：LLM上下文智能压缩引擎——在工具输出、日志、文件、RAG片段进入LLM之前自动压缩，节省60-85%上下文窗口。本周7月3日一天内连发v0.29.0和v0.30.0两个版本：新增Agent Persona功能（为编码/通用场景提供专属压缩策略，节省更多token），以及content-router智能分流（无损/有损压缩自动选择）。
解决什么：AI应用随着Agent长线任务增加，上下文窗口消耗呈指数增长——一次多步骤Agent任务可能消耗上百万token。Headroom透明插入压缩层，不改变应用代码即可大幅降本。新的Persona功能让压缩策略适配不同场景（编码vs写作vs客服），精准度更高。
适用场景：所有使用LLM API的应用（Agent框架、RAG、代码助手、客服机器人）；token消耗大、需要降本增效的生产环境；希望"无感接入"不需要改代码的团队。
怎么用：pip install headroom==0.30.0，在LLM调用链前加一行headroom.compress(content, persona="coding")，即可自动压缩。
🔥 本周密集发版：7月3日v0.30.0（Agent Persona）+ v0.29.0（content-router），一天两版
