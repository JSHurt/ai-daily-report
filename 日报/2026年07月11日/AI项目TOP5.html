Cline — VS Code 自主编程 Agent
⭐ ~64K
是什么：VS Code 里的开源自主编程 Agent 插件——直接读写项目代码、跑命令、调试错误，全程由你点"允许"确认。
解决什么：当下 AI 编程助手的模型绑定（Cursor 锁 OpenAI、Copilot 锁 OpenAI）和代码外传是开发者最大的两个顾虑。Cline 用统一接口把七大提供商给齐：Anthropic、OpenAI、DeepSeek、Google、Ollama、OpenRouter、Azure，想换就换。
适用场景：日常开发做功能加法、Bug 修复、重构；用本地 Ollama 模型做隐私安全的代码生成；重度编程用户跨模型对比效果。
怎么用：VS Code 商店搜 "Cline" 装插件 → 选模型提供商填 API Key → 描述需求 → Cline 自动规划并修改文件。
🔥 近期亮点：Sonnet5 全平台上线，七大模型提供商统一接入；持续活跃维护

OpenViking — Agent 上下文数据库
⭐ ~26K
是什么：字节跳动火山引擎开源的"上下文数据库"——把 Agent 的所有对话、决策、记忆结构化存储、按版本管理，专为 Agent 长期记忆设计。
解决什么：所有 LLM Agent 的共同痛点：会话窗口一关就失忆、用户的偏好和历史全清零。OpenViking 把它做成可插拔基础设施——Agent 接上就能"记住"用户，并能回溯历史版本、看清决策路径。
适用场景：C 端 AI 助手（"记住用户偏好"是基础体验）；企业 Agent 处理跨天的长流程任务（onboarding、审批、销售跟进）；需要"可审计"的多 Agent 协同。
怎么用：pip / npm 安装 openviking-sdk → 启动上下文服务 → Agent 通过 `viking.add()` 写入上下文、`viking.retrieve()` 检索历史；支持整站数据一键导入作为知识前置。
🔥 近期亮点：多版本上下文管理；整站数据导入；持续迭代中

Ollama — 本地一键运行大模型
⭐ ~140K+
是什么：本地运行大语言模型的轻量级开源工具——一条命令拉模型、一条命令跑起来，覆盖 Llama/DeepSeek/Qwen/Mistral/Gemma 数系列主流模型，自动处理量化和显存调度。
解决什么：开发者、企业内网用户、教育工作者三类人群普遍面临的痛点：依赖云端 API 又担心数据出境、订阅费月月涨、模型选择被锁定。Ollama 把"本地跑任意开源大模型"做成零配置体验，且自带 OpenAI 兼容 API——原本调用 OpenAI 的工具（Cursor/Cherry Studio/NextChat/各种 Agent 框架）改一行 base_url 就能切成本地。
适用场景：本地调试、隐私合规、为 Agent 提供统一本地推理后端、教学场景让学生离线体验 LLM。
怎么用：macOS：`brew install ollama`；Linux：`curl -fsSL https://ollama.com/install.sh | sh`；启动 `ollama serve`；拉模型 `ollama pull qwen3:32b`；调用 base_url=http://localhost:11434/v1。
🔥 近期亮点：模型库持续扩充；OpenAI 兼容 API 层修复；跨平台体验打磨

RAGFlow — 深度文档理解的 RAG 引擎
⭐ ~58K
是什么：开源的检索增强生成（RAG）引擎，专注"复杂格式文档"的深度解析——PDF/Word/Excel/PPT/图片中的表格、公式、图文混排都能精准切片，输入输出走统一工作流。
解决什么：普通 RAG 工具的"切块算法"对真实企业文档基本失效——合同里的表格被切成乱码、PPT 截图里的内容根本读不到。RAGFlow 的卖点是"读得懂复杂文档"：内置 OCR、表格识别、版面分析，把文档按"语义块"切而不是按字数切，混合检索（BM25 + 向量）× LLM 改写显著提升召回质量。
适用场景：法律、金融、医疗领域需要喂 PDF/合同/病历的私域知识库；企业把 Confluence/Notion/飞书文档建成可对话助手；Agent 接私域知识的统一检索层。
怎么用：docker compose 一键起服务；上传文档 → 自动解析/切片/embedding → 提问时混合检索 → 回答案 + 引用原文片段。
🔥 近期亮点：DeepDoc OCR 准确率持续优化；多栏论文识别；Agent 工作流节点增强

Pydantic AI — 类型安全的 Agent 框架
⭐ ~18K
是什么：用 Pydantic 风格写 AI Agent 的 Python 框架——类型注解约束输出结构，Agent 返回一定符合你定义的数据类型，再也不用写一堆 JSON 解析兜底代码。
解决什么：大模型输出格式不可控是工程化最大痛点——下游管道动不动报错。Pydantic AI 把 Pydantic 的类型安全直接带进 Agent：返回值会被自动校验，工具调用通过 `@agent.tool` 装饰器声明，多模态能力（图像、视频理解）也在持续扩展。
适用场景：Python 后端服务开发者、API 服务、数据处理管道、需要结构化输出的场景（合同抽取、报表解析、视频内容结构化）。
怎么用：pip install pydantic-ai；用类型注解定义数据模型；从主流提供商（OpenAI/Anthropic/Ollama）任选模型；`@agent.tool` 声明工具。
🔥 近期亮点：视频理解模型支持；多模态门槛降低；类型安全生态持续完善
