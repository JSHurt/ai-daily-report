ECC — Agent Harness 操作系统
⭐ 226,026
是什么：Agent Harness（鞍具）性能优化系统——自称"Agent Harness操作系统"。为 Claude Code、Codex、OpenCode、Cursor、Gemini CLI 等主流编码Agent提供统一的技能(skills)、本能(instincts)、记忆(memory)、安全(security)和研究优先(research-first)开发框架。本周7月4日提交大量更新：新增 Hermes Agent 和 OpenClaw harness 安装目标、新增 Kimi Code CLI 适配器、修复 issue 集群（关闭 #2295/#2298/#2303-#2306/#2340/#2408）。累计 22.6 万星、3.4 万 fork。
解决什么：不同编码Agent各自有一套CLI、配置、skill系统，切换成本极高。ECC将Agent本身抽象为"Harness"（类似操作系统对硬件的抽象）——无论是Claude Code还是Hermes还是Cursor，用户只需要在ECC中切换Agent harness，skill/记忆/安全策略无缝复用。从根本上解决了"Agent切换时一切重来"的痛点。
适用场景：同时使用多个编码Agent的开发者；团队希望统一管理Agent skill和记忆策略；需要跨Agent复用的安全审查（ecc-agentshield组件）场景。
怎么用：npm install ecc-universal，ecc install hermes/opencode/claude-code，配置后所有Agent共享 skill/instincts/memory。npm周下载量极高（ecc-universal + ecc-agentshield）。

Claude-Mem — Agent持久记忆系统
⭐ 85,832
是什么：为所有AI Agent提供跨会话持久记忆的开源系统。捕获Agent会话中的所有内容（对话、代码、决策），用AI自动压缩为结构化记忆，在后续会话中自动注入相关上下文。本周7月4日一天内连发 v13.10.0 和 v13.10.1 两个版本：修复 Codex hook 输出的 JSON 解析问题、保留空字符串 additionalContext、session-start 无效 JSON fallback。已支持 Claude Code/Codex/OpenCode/Cursor/Gemini CLI 等主流Agent。
解决什么：Agent每次新会话都是"失忆状态"——之前的长线任务进度、关键决策、踩过的坑全部丢失。claude-mem在Agent底层透明插入记忆层：每次会话自动记录→AI压缩→下次会话自动注入。相比简单的对话历史，claude-mem做的是语义级别的记忆压缩（不是存原文，而是提取关键知识点），让Agent真正做到"越用越聪明"。v13.10系列修复了 Codex 场景下的多个记忆注入 bug。
适用场景：需要Agent执行长线任务的场景（多日项目开发、持续代码审查、客户服务）；任何希望Agent"记住上一次做了什么"的团队；安全审计需要追溯Agent历史决策的场景。
怎么用：npm install -g @thedotmack/claude-mem@13.10.1，启动记忆服务（Chromadb后端），Agent自动记录和注入上下文。Web UI 可查看记忆图谱。

Graphify — 代码到知识图谱 Skill
⭐ 77,671
是什么：AI编码Agent的skill插件，将任意文件夹（代码、SQL schema、R脚本、Shell脚本、文档、论文、图片、视频）转换为可查询的知识图谱。支持 Claude Code、Codex、OpenCode、Cursor、Gemini CLI。YC S26入选项目。本周7月4日发布 v0.9.6：修复 Windows skill 名称兼容性、OpenCode 插件分隔符、文档语料库报告噪音（#1635/#1646/#1657）、增量检测 Office 文件源+长路径正确性（#1649/#1655/#1656）、JS/TS 跨文件调用门控（#1659，消除假跨包边）。
解决什么：大型代码库中Agent经常"找不到东西"——传统RAG只做文本检索，不理解代码的调用关系、依赖链、数据流。Graphify将代码按Leiden算法聚类为知识图谱节点，Agent可以通过图谱查询"这个函数被谁调用"、"修改这里会影响哪些文件"——让Agent真正理解代码结构而非仅做关键词匹配。
适用场景：大型代码库的Agent辅助开发（理解代码结构、重构影响分析）；复杂SQL schema的理解和查询生成；论文/文档的知识图谱化检索；多Agent协作时需要共享代码理解的场景。
怎么用：在 Agent skill 目录中安装 graphify skill（uv tool install graphify 或 npm），Agent 对话中说 "graphify this repo"，自动生成知识图谱并注入 Agent 上下文。

⭐ 79,452
是什么：把你的Agent团队组织成 7×24 全天候自动化运营——负责"雇佣"Agent、"排班"调度、"汇报"整个AI团队的产出。7月4日一天内密集发布 3 个 canary 版本（v2.2.10-canary.27/28/29）：新增 Upstash 工作流指标、暴露 run guard webhook 路由（供外部监控）、run card 展示运行最后消息、修复 gateway 模式下单 Agent @mention 路由。
解决什么：拥有多个Agent后最大的问题是"谁来管这些Agent"——何时执行哪个Agent、如何分配任务、如何汇报产出。LobeHub充当"CAO（Chief Agent Operator）"角色：Agent变成可雇佣的"员工"，用户只需设定目标和审批，CAO负责派活、调度、验收和日报。用户不用一直在线盯着。
适用场景：AI团队管理者——拥有多个Agent但不想手动调度的场景；需要Agent 24小时自动运转的电商/客服/运维；希望Agent产出日报并在必要时@用户审批的工作流。
怎么用：Docker部署 LobeHub，连接各Agent API，创建Workflow定义任务链，设置调度策略，Agent自动按班次执行。Web Dashboard 查看所有Agent运行状态和产出。
