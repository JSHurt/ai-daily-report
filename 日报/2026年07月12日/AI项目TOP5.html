mem0 — Agent 可插拔长期记忆层
⭐ ~38K
是什么：面向 AI Agent / LLM 应用的开源「记忆中间层」——以 SDK + 服务两种形态提供长期记忆的存取、检索、衰减与多用户/多会话隔离。
解决什么：几乎所有 LLM 应用都会撞上同一面墙：会话结束就失忆、用户偏好记不住、多轮上下文爆 token。mem0 通过「提取 → 存储 → 检索」三步把记忆做成可插拔中间件，替换 ChatHistory 中的 static prompt 拼接，配合向量库 + LLM 摘要，调用一次 API 即可让 Agent 跨会话记住用户。
适用场景：C 端 AI 助手（个性化陪聊、教育辅导、健身指导）；企业 Agent（CRM 跟进、客服回访、跨天审批）；多 Agent 协同中的共享记忆层。
怎么用：pip install mem0ai；初始化 memory 客户端 → 在每次对话结束时调 `memory.add()` 写入；下一次对话调 `memory.search(query)` 取回相关记忆自动拼入 prompt。
🔥 近期亮点：跨会话记忆准确率持续优化；多租户隔离；支持自定义 LLM / 向量库后端

vLLM — 高吞吐大模型推理引擎
⭐ ~32K
是什么：UC Berkeley 出品的开源大模型推理与服务引擎——以 PagedAttention（类似操作系统虚拟内存的分页机制）出名，把 GPU 显存利用率推到工业级 80%+，吞吐量比原生 HuggingFace Transformers 高 14–24 倍。
解决什么：推理贵、并发低、首 token 慢是私有化部署大模型的三大拦路虎。传统框架一次只能服务几个并发请求，显存大量碎片化浪费；vLLM 通过细粒度 KV cache 分页 + 连续批处理，把同一张 A100 / H100 服务几十路并发且 P99 延迟可控，自带 OpenAI 兼容 API，零代码迁移现有调用。
适用场景：AI 创业公司自建推理服务；大模型私有化部署（政企、医院、券商）；高并发在线 Agent 后端（同时服务上千用户）。
怎么用：pip install vllm；`vllm serve Qwen/Qwen3-32B --tensor-parallel-size 2` 启动服务；客户端按 OpenAI Chat Completions 协议对接 base_url=http://localhost:8000/v1。
🔥 近期亮点：Speculative decoding、Prefix caching、量化（AWQ/GPTQ）持续完善；多 LoRA 热加载

n8n — 可视化工作流 + AI Agent 编排
⭐ ~80K+
是什么：「可拼装的工作流引擎」——开源自托管的 Zapier 替代品，但多了代码节点、AI Agent 节点、HTTP/Webhook/数据库/Slack/Notion 等 400+ 集成，把任意系统的输入输出「可视化连」起来。
解决什么：业务方提需求、研发去对接系统、点数周写不完？n8n 把「触发 → 处理 → 条件分支 → AI 改写 → 写入」画成流程图，非工程也能搭。AI 用例：监听飞书消息 → 调 LLM 总结 → 写回 Notion；定时拉 GitHub Issue → Agent 分析 → 推企业微信。
适用场景：中小公司无人值守数据流（订单、客服、监控）；研发自建轻量 ETL / 告警；个人把碎片工具链成自动化中枢。
怎么用：docker run -d n8nio/n8n 或 npx n8n；浏览器访问 :5678；从节点面板拖拽「GitHub Trigger」「OpenAI」「Slack」等节点连线；导出 JSON 可做 CI/CD 复用。
🔥 近期亮点：原生 MCP（Model Context Protocol）节点；AI Agent 节点；社区 1万+ 模板

Firecrawl — 把任意网页变成 LLM-ready 数据
⭐ ~25K
是什么：开源「网页 → Markdown / JSON」爬取与结构化引擎——专为下游接 LLM 设计：自动处理 JS 渲染、反爬绕过、按 CSS 选择器或自然语言描述抓字段、批量整站抓取后输出干净的 LLM-readable 内容。
解决什么：RAG / Agent 项目 60% 的工程时间花在「把网页脏数据洗成模型能吃的格式」——传统爬虫 BeautifulSoup 在动态页面前失效、Scrapy 配置复杂、抽取规则脆弱。Firecrawl 一行 API 拿到结构化结果，配合 OpenAI / Ollama 还能让「自然语言描述抽取字段」取代手写 selector。
适用场景：RAG 知识库批量构建（电商详情页、行业资讯、技术博客）；Agent 联网搜索的数据采集层；企业内部系统对接（合规审计、竞品监控）。
怎么用：开源自托管 `docker compose up firecrawl` 或云服务 firecrawl.dev；调 `/scrape` 单页 / `/crawl` 整站；可选 `extract` 用 LLM 输出结构化 JSON。
🔥 近期亮点：JS 渲染稳定性；LLM Extract 自然语言抽取；批量并发提升

Cherry Studio — 多模型聚合桌面 AI 客户端
⭐ ~20K
是什么：国产生开源 AI 桌面客户端——一个窗口聚合 OpenAI / Anthropic / Gemini / DeepSeek / Qwen / Ollama / 各类国产大模型 API、内置知识库、Prompt 库、AI 绘画、MCP 服务发现；Win / Mac / Linux 全平台。
解决什么：普通用户在「ChatGPT 网页 + Claude 网页 + 文心一言 App + DeepSeek 网页 + 截图 OCR」之间反复切换，记不住 prompt、答完不知道存在哪。Cherry Studio 把多模型做成「一个窗口多 TAB」，本地 Markdown 知识库 + 跨服务对话历史，找到答案顺直接进笔记。
适用场景：个人把多模型当瑞士军刀（写作 / 翻译 / 编程 / 画图）；小团队统一调用多家模型做评估；学生 / 研究人员跨模型对比回答质量。
怎么用：GitHub Release 下载安装包 → 添加「服务商 API Key」或多模型 base_url → 任意模型开聊 → 拖入文档进「知识库」随时检索引用。
🔥 近期亮点：MCP 服务可视化发现；多模型对话历史合并视图；本地知识库增强
