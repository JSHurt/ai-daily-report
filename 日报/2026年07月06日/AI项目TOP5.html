Browser Use — AI Agent 的浏览器操控平台
⭐ 102,901
是什么：让 AI Agent 能够像人类一样操控网页浏览器的开源平台。本周7月1日发布 v0.13.3：全新 Browser Use CLI 3.0（基于 Browser Harness 引擎重写），新增 browser-use skill（让 Claude Code/Codex 等编码 Agent 一键安装浏览器操控能力），并固定 Browser Harness 依赖至 0.1.4。累计 10.3 万星。
解决什么：传统 Agent 调用网页只能通过 API，无法处理需要登录、填表、点击、翻页的真实网页交互。Browser Use 给 Agent 装上"手和眼睛"——Agent 能打开真实浏览器，看页面截图、理解DOM结构，然后点击按钮、填写表单、滚动翻页，完成人在浏览器里做的任何事。CLI 3.0 + skill 插件让这一切只需一句话即可启动。
适用场景：让 Agent 自动填报销单/注册账号/预约服务/爬取需要登录的动态页面；自动化测试 web 应用；Agent 替用户完成在线购物、订票等含多步交互的任务。
怎么用：pip install browser-use，然后在 Agent 中安装 browser-use skill，Agent 对话说"帮我登录XX网站提交订单"，Agent 自动打开浏览器执行操作链。
🔥 本周里程碑：7月1日 CLI 3.0 + browser-use skill，Agent 可一键安装浏览器操控能力

解决什么：编码 Agent 最大的隐性成本是 token 消耗——Agent 在对话中输出的解释、描述、确认信息占了大量 token（经常比代码本身还多）。Caveman 把 Agent 的输出风格改成极简模式："use few token when few token do trick"——能5个词说完的绝不用50个词。实测 7.5 万 token 的任务只用 2.7 万。v1.9.1 重点修复安装安全问题：curl|bash 安装不再从 main 分支拉取（改为不可变 release tag），安装后校验文件完整性。
适用场景：任何使用 Claude Code/Codex/OpenCode 的开发者；长期项目高频 Agent 交互场景（token 账单敏感）；团队统一配置 Agent 话术风格。
怎么用：在 Claude Code 的 skills 目录安装 caveman skill，Agent 自动切换为极简输出模式。npm 全局安装或 curl|bash 一键装。
🔥 本周安全加固：7月3日 v1.9.1，50变更集中在安装安全（锁定release tag+完整性校验）+诚实报告节省率

Claude-Mem — Agent 跨会话持久记忆
⭐ 85,984
是什么：为所有 AI Agent 提供跨会话持久记忆的开源系统——Agent 每次对话自动记录、AI 压缩为结构化记忆、下次会话自动注入相关上下文。支持 Claude Code/Codex/OpenCode/Cursor/Gemini CLI 等主流 Agent。本周持续活跃更新（7月5日 pushed），此前 7月4日连发 v13.10.0 → v13.10.1 修复 Codex hook 的记忆注入缺陷。累计 8.6 万星。
解决什么：Agent 每次新会话都是"失忆"状态——之前的任务进度、踩坑经验、项目上下文全部丢失。claude-mem 在 Agent 底层透明插入记忆层：语义级压缩（不是存原文而是提取知识点），让 Agent "越用越聪明"。比如昨天的重构决策、上周修的 bug 根因，新会话的 Agent 能自动回忆。
适用场景：需要 Agent 执行多天长线任务的开发场景；持续代码审查需要追溯历史决策；安全审计需要查看 Agent 的历史行为和决策记录。
怎么用：npm install -g @thedotmack/claude-mem@latest，启动记忆服务（Chromadb 后端），Agent 自动记录和注入上下文。Web UI 可查看记忆图谱。
🔥 本周持续打磨：7月4日同日双版（v13.10.0/v13.10.1）修复 Codex hook 记忆注入，累计 8.6 万星
