01
OpenCode — 终端原生 AI 编程 Agent
⭐ ~182K
是什么：开源终端优先（terminal-first）的 AI 编程 Agent——Go 写的单一二进制，跨平台无依赖，启动即工作；走 Claude / GPT / Gemini / 本地 Ollama 全模型，认 Claude Code 那套 /command 交互但完全开源可私有部署。
解决什么：VSCode Copilot 只能在 IDE 里补全；Cursor 锁定工作区；Claude Code CLI 虽然强但绑死供应商。开 Agent 想在 SSH 远程服务器、Docker 容器、CI runner 里复用同一套「能用自然语言驱动代码修改」的能力时，多数工具都掉链子——OpenCode 把「把 Agent 装进任意 shell」做成一行 go install 的体验，配置走 ~/.config/opencode，session 文本可共享可回放。
适用场景：需要在远程服务器长期跑 Agent 做运维脚本、批改、研究代码；要做 IDE 无关的「CLI 编程」工作流的个人/小团队；想在 CI 里集成「让 AI 顺手修 lint 错」的工程团队。
怎么用：curl 安装二进制 → `opencode` 进入交互 shell；`/init` 把「仓库说明 + 习惯」喂给 Agent；用自然语言描述改动，Agent 直接改文件、跑命令、回贴 diff；`.opencode.json` 配置默认供应商与 model。
🔥 近期亮点：全模型兼容（Anthropic / OpenAI / Gemini / Ollama / 国产模型）；session 可回放；TUI / WebUI 双前端

OpenHands — 自主开发 AI Agent
⭐ ~80K
是什么：原名 OpenDevin、当前叫 OpenHands 的「自主软件工程 Agent」平台——给 Agent 配完整沙箱（shell + 编辑器 + 浏览器），接一个 Issue 起手就能从克隆仓库→改代码→跑测试→提 PR 全链路搞定；自带 Web UI + CLI + GitHub App 三种使用姿态。
解决什么：补全型 Copilot 只能帮写函数，整仓库级任务（重构、迁移、加端到端测试、修一个跨文件 bug）它接不住。OpenHands 让模型像一个真的初级工程师：进容器、自己 grep、自己 git diff、看到红测试自己调；30 多个内置 Agent 类型覆盖 code-review / swe-bench / research，可自定义 Agent 模板。
适用场景：中小团队把「issue → PR」自动化，对成熟仓库提速 review-backlog；SWE-bench 类评测的基线；想做「私有化 Claude-Code-with-sandbox」的企业内 Agent 平台。
怎么用：Docker 一键起后端 + 前端；Docker Hub 镜像含 sandbox runtime；接入 GitHub App 后，issue 评论里 `/openhands` 触发 Agent，自动开 PR；也可 `openhands` CLI 直接喂本地任务。
🔥 近期亮点：内置 SWE-bench 高分 Agent；自定义 Agent SDK；Docker sandbox 默认隔离

Dify — LLM 应用编排与 RAG 平台
⭐ ~148K
是什么：「生产级 Agentic workflow 开发平台」——开源版 Langflow + FastAPI + 知识库一体化：可视化拖拽搭 prompt 流、工具调用、条件分支、知识库检索、Agent；内置模型路由、观测、日志、用户与权限、API 与 Web App 一键发布。
解决什么：业务团队想做「公司知识库问答」「多步自动化工作流」「带 RAG 检索的内部 Copilot」，要么写一堆 LangChain 代码、要么硬上 Coze / 扣子这类闭源。Dify 把「可视化画流程图 + 直接接 LLM/RAG/工具」做成开箱即用的产品，B 端落地开箱即用，可私有化、可二次开发。
适用场景：企业内 AI 助手（HR 问答、IT Helpdesk、销售话术）；客服 / 营销 / 法务等单部门 AI 工作流；想把工作流编排思路快速验证 MVP 而不想重写的创业团队。
怎么用：Docker Compose / 云 SaaS 启动；从模板库拖一个「知识库 + LLM + 工具」应用 → 接入企业文档 → 调 LLM 供应商 API Key → 发布 WebApp / API；后续工作流改动在 UI 上拖拽即可。
🔥 近期亮点：Agentic workflow + 多 Agent 协作；可观测与 token 计费；私有化部署成熟

Ollama — 一键本地大模型推理
⭐ ~175K
是什么：「本机的模型管家」——把 Llama / Qwen / GLM / DeepSeek / gpt-oss / Kimi-K2.6 / MiniMax 等几十种开源大模型（GGUF 量化）封装成一行命令就能跑：自动下权重、按显存挑量化档、起 OpenAI 兼容 API 服务；Mac（Apple Silicon）/ Linux / Windows 全平台。
解决什么：开源模型一抓一大把，但「量化格式怎么选 / 显存不够怎么办 / 怎么开 OpenAI 兼容 API / 怎么多模型切换」这一堆对接工程让个人开发者放弃 90%。Ollama 用 Modelfile + 模型市场一抹平，把「我就要本地跑某个 Qwen3-Coder」变成 `ollama run qwen3-coder:32b` 一句话。
适用场景：隐私敏感场景（律师 / 医生 / 研究者本地用大模型）；离线开发（飞机上、保密机房）；Agent 框架的本地模型后端（OpenCode、Continue、Cherry Studio 全部接 Ollama）；CI 里跑轻量化评审模型。
怎么用：官网装客户端 → `ollama pull qwen3:32b`；`ollama run qwen3:32b` 进 chat；如要把本地当 OpenAI API 服务，启动后访问 `http://localhost:11434/v1`，任何 OpenAI 兼容客户端直接用。
🔥 近期亮点：原生支持 Kimi-K2.6 / GLM-5.1 / gpt-oss 等新模型；多模态视觉模型；多 GPU 并行

05
browser-use — 让 Agent 自己「上网操作」
⭐ ~103K
是什么：面向 AI Agent 的「浏览器自动化 SDK」——Python 一行代码，让大模型看到 DOM 结构（不只是截图）+ 用自然语言「点 / 输 / 选 / 翻页」驱动 Chromium；和 Playwright / Selenium 不同，它把浏览器状态喂成「Agent 友好的可读文本」，模型直接复用普通聊天能力就能操作网页。
解决什么：大多数「让 AI 自己上网」的方案要么靠截图（贵、慢、错率高），要么要写一坨 CSS/XPath 选择器（脆、跨站失效）。browser-use 把 DOM 树 / 可点击区域 / 表单字段抽象成「模型能直接看懂的清单」，背后用 Playwright 做执行，一行 `await agent.run("去 X 网站订会议")` 就能跑——本研究里 90%+ 网站自动化成功率。
适用场景：研究 Agent 自动查资料（论文、产品比价、机票）；运营自动化（填表、抓后台数据）；AI 产品里需要「有手」的嵌入式 LLM 工作流；对接老旧系统没 API 只能走网页的情形。
怎么用：pip install browser-use；`from browser_use import Agent`；`agent = Agent(task="帮我把 linkedin 上今天 10 条 AI 帖子点赞", llm=ChatOpenAI(model='gpt-4o'))`；`await agent.run()`——Agent 自启浏览器、看 DOM、点按钮。
🔥 近期亮点：DOM 优先 + 截图兜底的双模式；多标签页并行；与 LangChain / LlamaIndex 原生集成
