OpenClaw — 跨平台个人 AI 助手
⭐ ~383K
是什么：在你自己设备上跑的「个人 AI 助手」——Gateway 做控制面，产品形态是「常驻助手」：接 WhatsApp / Telegram / Slack / Discord / 飞书 / 企业微信 / QQ / Teams 等你已经在用的通道；支持 macOS/iOS/Android 语音对话，还能渲染可交互的 live Canvas。
解决什么：ChatGPT/Claude 网页是「你去找它」；企业 Bot 往往绑死供应商或内网审批。OpenClaw 反过来：助手住在你设备上，消息从你日常 IM 进来，回复也回同一通道——单用户、本地优先、随时在线，而不是再开一个新 App 聊天窗口。
适用场景：想把 AI 嵌进日常沟通流的个人/小团队；需要跨端（手机+电脑）常驻助手；对「数据走自己设备、通道自己控」有要求的自托管用户。
怎么用：按官方 Getting Started 安装 Gateway → 绑定至少一个消息通道与模型供应商 → 在 IM 里直接对话；更新走官方 release（本周已发 v2026.7.1）。详见 docs.openclaw.ai。
🔥 近期亮点：v2026.7.1（7/13）；多通道 IM 原生接入；自设备常驻 + live Canvas

解决什么：本地跑了 Ollama / vLLM / LocalAI 之后，缺的往往不是模型，而是「家人同事也能点开就用」的界面：多会话、知识库上传、工具/插件、模型切换、权限隔离。Open WebUI 把「自托管 ChatGPT 体验」做成开箱即用，而不是再写一坨 Gradio 原型。
适用场景：家庭实验室/小团队统一聊天入口；企业内网要一套可审计的多模型前端；把 Ollama 从「命令行玩具」升级成「全员可用服务」。
怎么用：`docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui ghcr.io/open-webui/open-webui:main`（或官方 `:ollama` / `:cuda` 镜像）→ 浏览器打开 → 配置 Ollama 地址或 OpenAI 兼容端点 → 上传文档开 RAG。

Headroom — Agent 上下文压缩中间层
⭐ ~59K
是什么：面向 AI Agent 的上下文压缩层——在工具输出、日志、文件、RAG chunk 进入 LLM 之前做内容感知压缩；提供 library / proxy / MCP 三种接入，本地优先、可逆；宣称 JSON 类数据可少 60–95% Token，编码 Agent 场景约 15–20%。
解决什么：Agent 越能干，上下文越容易被「一次 cat 大文件 / 一次跑测日志 / 一坨 RAG 召回」灌满——结果是贵、慢、还更容易丢重点。Headroom 不改你的 Agent 逻辑，只在进模型前把噪声压掉，让同样的上下文窗口装更多有效信息，账单和延迟一起降。
适用场景：Coding Agent / 工具调用密集的工作流；RAG 管道里 chunk 过胖的应用；想用 MCP 给现有 Claude Code / Cursor / 自研 Agent 统一加「省 Token 层」的团队。
怎么用：`pip install headroom-ai` 或 npm 装同名包 → 以库调用压缩函数，或以 proxy/MCP server 挂到 Agent；配置 content-aware compressor，对 JSON/日志/代码输出走不同策略（详见官方 docs）。
🔥 近期亮点：v0.31.0（7/9）；library + proxy + MCP 三形态；JSON 最高省 95% Token
