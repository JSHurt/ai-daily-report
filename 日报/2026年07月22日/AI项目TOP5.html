LiteLLM — 统一多模型 AI 网关
⭐ 54.3K
是什么：轻量高速 AI Gateway——Rust 核心 + Python SDK，用 OpenAI 兼容（或原生）格式统一调用 100+ LLM API，内置成本追踪、护栏、负载均衡与日志，覆盖 Bedrock / Azure / Anthropic / Vertex / vLLM / NIM 等；最新预发 v1.94.0-rc.2，稳定线 v1.93.0。
解决什么：业务里模型供应商一多，SDK、鉴权、限流、账单和故障切换就变成运维泥潭。LiteLLM 把「一个入口吃下所有模型」做成可部署网关，应用只对接一种协议，后面随时换模型或加 MCP 网关能力。
适用场景：多云/多厂商模型统一接入；需要按量计费与护栏的中台；给 Agent/工作流平台做后端路由；自建 OpenAI 兼容代理。
怎么用：Docker/源码部署 Proxy → 配置模型与密钥 → 客户端改 base_url 指向网关；今日仓库仍有 UI/依赖推送（sharp 升级、护栏测试 mock 修复）。GitHub API 实时：2026-07-22 推送。
🔥 近期亮点：54.3K⭐；v1.93.0 / v1.94.0-rc.2；今日 UI 与依赖连更；Docker cosign 签名校验

Langflow — 可视化构建与部署 AI Agent
⭐ 152.2K
是什么：面向 Agent 与工作流的可视化搭建/部署工具——拖拽组件连成多 Agent、RAG、工具调用流水线，可本地或服务端发布；正式版 v1.10.2，主线已有 v1.11.0 标签与本周发版流水线修复，仓库今日仍有推送。
解决什么：纯代码搭 Agent 迭代慢、难给产品/运营共建。Langflow 把模型、检索、工具、记忆做成节点，让「能跑的 Agent」从画布直接部署，缩短从原型到可演示链路。
适用场景：产品/算法共建 Agent 原型；企业内 RAG 与多 Agent 演示；需要快速换模型与全局变量覆盖的场景。
怎么用：按 langflow.org 安装（pip/Docker）→ 画布组装流程 → 发布 API；v1.10.2 含全局变量模型覆盖、telemetry 会话修复、默认关闭 mem0 遥测等。GitHub API 实时：2026-07-22 推送。

Graphify — 代码与文档一键变知识图谱
⭐ 93.1K
是什么：面向 Coding Agent 的 /graphify 技能与工具链——把代码库、文档、SQL schema、配置与 PDF 转成可查询知识图谱：本地确定性 AST 解析、每条边可解释、不依赖向量库；最新 v0.9.23（7/21），仓库已 bump 向 0.9.24。
解决什么：纯向量 RAG 对「谁调用谁、改一处影响哪」不敏感。Graphify 用图结构回答调用链、影响面与邻居节点，给 Claude Code / Cursor / Codex / Gemini CLI 更准的代码上下文。
适用场景：大型仓库让 Agent 先建图再改代码；影响分析与 code search；无向量库的本地知识增强；MCP 查询图邻居。
怎么用：在支持的 Agent 环境安装 graphify skill → 对仓库执行 graphify 构建/更新 → 用 explain/affected/query 提问；v0.9.23 修正 calls 边调用点行号展示（此前误指函数定义行）。推送：2026-07-21。
